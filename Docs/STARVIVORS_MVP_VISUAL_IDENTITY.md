# STARVIVORS MVP VISUAL IDENTITY

This file defines the MVP visual direction.

## Visual Goal

STARVIVORS should read as a clean arcade sci-fi survival game.

The MVP should prioritize:

- Readability
- Strong silhouettes
- Clear danger zones
- Simple geometric shapes
- Satisfying particles
- Fast visual comprehension during movement

## Style Direction

Use simple shapes first.

Recommended MVP visual language:

- Player ships as readable geometric silhouettes
- Enemies as distinct simple shapes
- Asteroids and debris as rough polygon shapes
- Black holes as high-contrast gravity hazards with clear range indicators
- Projectiles as bright, readable shapes
- Pickups as small but distinct resource icons or shapes
- Particles for impact, thrust, scrap collection, and destruction feedback

The MVP does not need complex art assets.

## Readability Rules

Gameplay clarity wins over decoration.

Every active object must be readable at gameplay speed:

- Player location must always be obvious
- Enemy types must be distinguishable
- Hazard debris must not look like harmless background decoration
- Black hole pull and kill zones must be visually clear
- Pickups must stand out without competing with hazards
- UI must not hide incoming danger

## Black Hole Visual Rules

Black holes are a signature feature, not background decoration.

MVP black holes should have:

- A visible center
- A clear danger or consumption zone
- A visible gravity influence range
- Motion that is readable against the starfield
- Visual feedback when consuming entities

Do not make black holes visually subtle.

## Background Rules

The arena should feel like open space.

Use a dark starfield or generated space background. Avoid busy backgrounds that hide ships, bullets, debris, pickups, or black hole ranges.

## WIP Visual Rules

WIP content may be visible in menus or previews if useful, but it must be clearly tagged.

WIP content must not look broken, selectable by accident, or fully implemented if it is not active gameplay content.
